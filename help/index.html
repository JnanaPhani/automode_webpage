Guidance for configuring Epson vibration and IMU sensors in Auto Start mode

Common Troubleshooting

Browser can’t find the device

  • Unplug the USB cable, wait 5 seconds, and reconnect it firmly.
  • Click “Select Sensor” again and choose the device from the Chrome/Edge picker.
  • On Windows, confirm no other application (e.g., serial terminal) is using the port.
  • On Linux, ensure your user is in the dialout group and reconnect the cable.

Configuration stops with an error

  • Power-cycle the sensor and try the configuration again.
  • Confirm you selected the correct sensor profile before starting.
  • Try a lower baud rate (230400) if the connection is unstable.

Choosing an IMU sampling rate

  • Faster rates (2000 / 1000 sps) require higher UART bandwidth—stick to 921600 or 460800 baud respectively.
  • The portal auto-sets the baud rate to 921600 when you pick those high rates. If you override it, heed the warning banner before proceeding.
  • Slower rates (250 sps and below) apply more averaging; expect smoother but slower data.
  • The portal programs the recommended moving-average filter taps automatically—no manual filter entry is needed.

Still seeing red status banner?

  • Restart the sensor and repeat the configuration steps.
  • Close other browser tabs that might be talking to the same device.
  • Reach out to Zenith Tek support with the log output for deeper analysis.
